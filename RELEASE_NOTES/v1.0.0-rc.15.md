### Emission Calculation Fixed

Fixed a significant bug in the 7-day and 30-day emission rate calculations that was causing **~10% underreporting** of daily TAO emissions.

#### The Problem
- `emission_7d` showed **6,517 TAO/day** instead of the actual **~7,184 TAO/day**
- Root cause: Data gaps in the first days of issuance history collection (Nov 23-24)
- One day with only partial data showed ~1,432 TAO/day instead of ~7,180, pulling down the average
- This caused halving projections to be off by ~2 days

#### The Fix
- Added **standard deviation check** (threshold: 500 TAO/day) to detect unreliable data
- If SD exceeds threshold â†’ automatically fallback to more recent, cleaner data periods (4d/3d/2d)
- Applied same logic to `emission_30d` (requires >= 14 days of clean data before using 30d average)
